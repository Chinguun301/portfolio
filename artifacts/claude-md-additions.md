## Babysitter

This project is set up with [babysitter](https://github.com/a5c-ai/babysitter) — an AI orchestration framework.

### Project Profile

- **Project**: Portfolio — Modern personal portfolio for a Frontend Developer
- **Stack**: Next.js 16, Tailwind CSS 4, Framer Motion, TypeScript, i18n (EN/MN)
- **Deploy**: Vercel
- **Workflow**: Trunk-based, manual releases, Conventional Commits
- **Autonomy**: Semi-autonomous

### Recommended Commands

| Command | Description |
|---------|-------------|
| `babysitter run:create ...` | Start a new babysitter orchestration run |
| `babysitter run:iterate .a5c/runs/<runId>` | Continue orchestrating a run |
| `babysitter run:status <runId>` | Check run status |
| `babysitter task:list <runId> --pending` | List pending tasks |

### Installed Processes

- `cradle/project-install` — Project onboarding and setup

### Recommended Skills

- `rpiv-pi:frontend-design` — UI/UX design and analysis
- `rpiv-pi:design` — Design complex features with vertical slices
- `rpiv-pi:blueprint` — Plan multi-component features
- `rpiv-pi:research` — Research UI/UX patterns and solutions
- `rpiv-pi:explore` — Explore solution options
- `rpiv-pi:commit` — Create structured git commits
- `rpiv-pi:code-review` — Review code changes
- `rpiv-pi:changelog` — Keep changelog updated

### Methodology

Use **iterative-convergence** methodology: iterate on features with quality gates, converge toward the goal with each cycle.
