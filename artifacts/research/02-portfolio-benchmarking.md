## 1. Brittany Chiang (brittanychiang.com)

**Source**: https://brittanychiang.com/

### Profile
Senior Frontend Engineer at Klaviyo. One of the most-cited developer portfolios in the industry.

### Strengths
- **Clean, minimal layout** with excellent typographic hierarchy
- **Accessibility-first**: Skip to content link, semantic HTML, focus management
- **Content sections**: About → Experience → Projects → Writing (blog)
- **Project cards** with clear tech stack tags and links (GitHub, live demo)
- **Accessibility focus**: Dedicated section about accessibility + a11y role
- **"Loosely designed in Figma and coded in VS Code"** — relatable designer-developer ethos
- **Footer with tech stack credits**: What the site is built with (Next.js, Tailwind, Vercel)

### Patterns That Convert
- "Skip to Content" — accessible and professional
- Clear role title + company for each experience entry
- Project cards show: screenshot, title, description, tech stack, links
- Writing section demonstrates thought leadership
- Résumé download link in navigation
- Social links (GitHub, LinkedIn, CodePen, Instagram, Goodreads) show personality

### Relevance
**High**. This is the gold standard for developer portfolios. Imitate the structure, differentiate with visual design and personal brand.

## 4. Industry Portfolio Patterns

### Information Hierarchy (Most Effective)
1. **Hero**: Name, role, one-line value prop, CTA buttons
2. **About**: Who you are, what you do, what you care about
3. **Experience (timeline)**: Roles, companies, key achievements
4. **Featured Projects**: Best work, with screenshots and tech stack
5. **Skills/Tech**: Tools and technologies (compact format)
6. **Writing/Blog**: Technical articles (optional but valuable)
7. **Contact**: Email, social links, résumé

### Patterns That Convert Visitors → Offers
| Pattern | Effectiveness | Implementation |
|---------|--------------|----------------|
| Clear value prop in hero | Critical | "I build accessible, pixel-perfect experiences for the web" |
| Project case studies | High | Screenshot → Problem → Solution → Tech → Links |
| Social proof | High | Client logos, testimonials, GitHub stars |
| Résumé download | Medium | PDF in nav or footer |
| Blog/Writing | Medium | Demonstrates expertise, improves SEO |
| Newsletter/CTA | Low-Medium | Optional, depends on audience |
| Contact form | Low | Email link is often sufficient |

### 2026 Trends in Developer Portfolios
- **AI-assisted content**: AI-generated project descriptions reviewed by developer
- **Interactive demos**: Embedded code playgrounds (CodeSandbox, StackBlitz)
- **Dark mode default**: Most developer audiences prefer dark theme
- **Performance as feature**: Lighthouse scores, 100/100 badges, fast load times
- **Minimal JavaScript**: Reduced bundle size, progressive enhancement
- **GSAP/Framer Motion animations**: Subtle scroll-triggered reveals
- **Terminal/CLI aesthetic**: Glowing text, ASCII art (trendy but divisive)
- **Micro-interactions**: Button hover states, link underlines, smooth page transitions

## 5. Portfolio Design Anti-Patterns (to Avoid)

| Anti-Pattern | Why It Fails |
|-------------|--------------|
| Too many animations | Distracts from content, hurts performance |
| Heavy hero with particle effects | Looks dated, developer cliché |
| "Under construction" badges | Unprofessional |
| Auto-playing media | User-hostile |
| Inaccessible color contrast | Excludes users, looks amateur |
| Stock photos of developers | Inauthentic |
| Timeline that stops years ago | Looks abandoned |
| Broken links or 404 pages | Loses trust immediately |
| No mobile responsiveness | Unacceptable in 2026 |
| Slow load times | First impression ruined |
